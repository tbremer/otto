# Question Tool Patterns

## Structure

Every `question` tool call should include:
- **header**: Short context label (e.g., "Next Steps", "Recovery Options")
- **question**: The actual question being asked
- **options**: 2-5 choices, each with optional description

## Option Guidelines

1. **Lead with action verb**: "Create plan", "Review files", "Skip task" (not "Plan creation", "File review")
2. **Include descriptions for non-obvious options**: Skip for simple yes/no
3. **Order by likelihood**: Most common choice first
4. **Always include escape hatch**: "Cancel", "Skip for now", "I'll decide later"

## Common Patterns

### Confirmation (yes/no with context)
```
header: "Confirm Action"
question: "Ready to proceed with {action}?"
options:
  - "Yes, proceed"
  - "No, cancel"
```

### Multiple choice (pick one)
```
header: "Choose Option"
question: "{What decision needs to be made}?"
options:
  - label: "Option A"
    description: "{What this does}"
  - label: "Option B"
    description: "{What this does}"
  - label: "Skip for now"
```

### Recovery (after error)
```
header: "⚠ {Error Type}"
question: "How would you like to proceed?"
options:
  - "Fix and retry"
  - "Skip this step"
  - "Abort"
```

### Next steps (after completion)
```
header: "Next Steps"
question: "What would you like to do next?"
options:
  - label: "{Primary action}"
    description: "{Why you'd do this}"
  - label: "{Secondary action}"
  - label: "Done for now"
```

## Anti-patterns

- Questions without options (use conversational text instead)
- More than 5 options (split into categories or simplify)
- Options without clear action ("Maybe", "Not sure")
- Missing escape hatch (user should always be able to exit)
